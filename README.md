# Запуск и начальная настройка проекта

## Полный запуск проекта

В настоящее время общий docker-compose не работает, поэтому для запуска проекта необходимо запускать компоненты по отдельности:

### 1. Подготовка и запуск базы данных и объектного хранилища

Сначала запустите Postgres и MinIO с помощью docker-compose в директории backend:

```bash
cd /workspace/backend
docker-compose up -d
```

Это запустит:
- PostgreSQL с pgvector для хранения данных
- MinIO для объектного хранения

### 2. Запуск ML/LLM компонента

Перед запуском убедитесь, что загрузили модели с помощью скрипта `download.ipynb` в директории `/workspace/ml/llm`:

```bash
cd /workspace/ml/llm
# Запустите Jupyter notebook или выполните команды из download.ipynb для загрузки моделей
python run.py
```

### 3. Запуск бэкенда

Установите зависимости и запустите бэкенд:

```bash
cd /workspace/backend
pip install -r requirements.txt
python main.py
```

### 4. Запуск фронтенда

Установите зависимости и запустите фронтенд:

```bash
cd /workspace/frontend
npm install
npm run dev
```

После этого проект будет доступен по адресу http://localhost:5173

## Описание реализованных функций

В проекте реализовано:

- **Аутентификация и авторизация пользователей** - регистрация, вход, управление профилем
- **Административная панель** - интерфейс для управления пользователями, задачами и статистиками
- **Работа с задачами** - создание, редактирование и управление задачами через админ панель
- **Обработка естественного языка с использованием LLM** - интеграция с языковыми моделями для анализа и генерации текста
- **Хранение файлов** - использование MinIO для хранения медиафайлов и документов
- **Асинхронная обработка** - использование очередей сообщений для фоновой обработки задач
- **API интерфейсы** - RESTful API для взаимодействия между компонентами системы
- **Работа с векторными данными** - использование pgvector для хранения и поиска векторных представлений
- **Генерация и синтез речи** - реализация TTS (Text-to-Speech) функциональности в ML компоненте
- **Парсинг задач** - автоматический анализ и обработка задач с использованием ML
- **Управление пользователями** - возможность назначения ролей, включая права администратора
